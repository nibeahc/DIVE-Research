# Skill: write-coverletter

## 목적

자소서 문항과 글자수를 입력받아
해당 사용자의 어투와 경험을 바탕으로 문단을 작성한다

## 트리거

"[이름] 자소서 써줘: [문항] / [글자수]"

## 글자수 규칙

- 글자수가 주어지지 않으면 작성 전에 반드시 먼저 물어본다
- 띄어쓰기 포함/제외를 명시하지 않으면 **기본값은 띄어쓰기 포함**으로 처리한다
- 글자수 초과는 절대 금지. 출력 후 실제 글자수를 반드시 표기한다

## 실행 순서

### 1. 사용자 데이터 로드

- users/[이름]/writing-style.md 읽기
- users/[이름]/index.md 읽기
- users/[이름]/관심직무.md 읽기

### 2. 직무 파악

트리거에 직무명이 포함된 경우 → 관심직무.md 에서 해당 직무의 핵심 키워드를 읽어 경험 선택에 활용한다
트리거에 직무명이 없는 경우 → 관심직무.md 의 직무 목록을 보여주고 어떤 직무로 작성할지 선택하게 한다

### 3. 문항 분석

문항의 핵심 의도와 키워드를 파악한다

예:

- "갈등을 극복한 경험" → 키워드: 갈등, 극복, 팀워크
- "지원 동기" → 키워드: 가치관, 동기, 직무역량
- "본인의 강점" → 키워드: 강점, 성격, 성과

### 4. 경험 후보 탐색

index.md 의 카테고리/키워드에서 문항과 관련된 경험을 찾는다
관련 경험이 여러 개면 가장 구체적이고 임팩트 있는 것을 선택한다
선택한 경험 파일을 users/[이름]/experiences/ 에서 읽는다

**경험 선택 원칙:**

- 활동은 **최대 2개, 기본 1개**만 사용한다
- 여러 활동을 나열하는 방식은 금지. 하나의 경험을 깊이 있게 풀어낸다
- 2개를 쓸 경우 두 경험이 자연스럽게 연결되어야 한다

### 5. 문단 작성

아래 기준을 반드시 지킨다

- writing-style.md 의 어투와 문체를 따른다
- 경험의 STAR 구조를 자연스럽게 녹여낸다
- 문항의 의도에 맞게 강조점을 조절한다
- 글자수 이내로 맞춘다 (띄어쓰기 포함/제외 기준 적용)
- 문단 형식으로 작성한다 (bullet point 사용 금지)
- 경험을 지어내거나 과장하지 않는다

### 6. 출력 형식

```text
---
[문항]

[작성된 문단]

참조한 경험: [경험명] ([파일명])
글자수: [N]자 (띄어쓰기 포함/제외)
---
```

### 7. output 저장

작성 완료 후 outputs/[이름]/[기업명]_[직무명].md 에 저장한다

저장 형식:

```text
# [기업명] [직무명]

## [문항 전문]
글자수: [N]자 (띄어쓰기 포함/제외)

[작성된 문단]

---
참조한 경험: [경험명]
작성일: YYYY.MM.DD
```

- 같은 파일이 이미 있으면 새 문항을 하단에 추가(append)한다
- 기업명과 직무명은 사용자가 말한 것 그대로 사용한다 (예: BGF리테일_영업관리.md)

## 절대 금지

- 경험을 지어내거나 과장하지 말 것
- writing-style.md 와 다른 어투 사용 금지
- 글자수 초과 금지
- bullet point 형식 사용 금지
- 활동을 3개 이상 나열하는 것 금지
