> 작성일: 2026-03-31

---

## 이 프로젝트가 뭔가

자소서 자동 작성 툴.
여러 사용자의 자소서 원본을 파싱해서 경험 DB를 만들고,
문항과 글자수를 던지면 해당 사용자 어투로 문단을 자동 작성한다.
이력 조회 기능도 포함.

## 각 파일 핵심 요약

### CLAUDE.md

- 프로젝트 전체 운영 규칙
- 폴더 구조, 명령어, 절대 금지 사항 정의
- 새 사용자 추가 방법 포함

### templates/experience-entry.md

- 경험 파일의 기본 형식
- 메타 (시기/기관명/역할/카테고리/키워드)
- 활동 내용 요약 (이력 조회 시 표에 표시)
- STAR 구조 (자소서 작성 시 참조)
- 이 경험으로 쓸 수 있는 문항 유형

### skills/parse-experiences.md

- 트리거: "[이름] 파싱해줘"
- raw/ 전체 읽기 → writing-style.md 생성
- 경험 전체 목록 먼저 추출 → 중복 판단 → 파일 생성
- 중복 확실: 자동으로 합침 (가장 풍부한 표현 선택)
- 중복 애매: 별도 파일 + ⚠️ 표시
- 시기 불명확: 0000-[활동명].md + "확인 필요" 표시
- 완료 후 파싱 결과 보고 출력

### skills/write-coverletter.md

- 트리거: "[이름] 자소서 써줘: [문항] / [글자수]"
- 글자수 없으면 먼저 물어봄 (띄어쓰기 포함/제외 여부도)
- writing-style.md 어투 반드시 따름
- 문단 형식으로 작성 (bullet point 금지)
- 출력 하단에 참조한 경험 명시

### skills/query-experiences.md

- 트리거: "[이름] 이력 보여줘" / "[이름] [키워드] 관련 이력 보여줘"
- 표 형식 출력 (활동명 / 기간 / 기관명 / 역할 / 활동내용 요약)
- 시기순 정렬
- 확인 필요 항목은 표 하단 별도 표시

## 다음에 할 것

1. **Claude Code 설치**

   ```bash
   npm install -g @anthropic-ai/claude-code
   ```

2. **프로젝트 폴더에서 Claude Code 실행**

   ```bash
   cd ResumeProject
   claude
   ```

3. **사용자 폴더 만들고 자소서 넣기**

   ```text
   users/[이름]/raw/ 에 자소서 파일 넣기
   ```

4. **파싱 테스트**

   ```text
   [이름] 파싱해줘
   ```

5. **파싱 결과 검토**

   - 확인 필요 항목 (시기 불명확, 중복 애매) 직접 확인
   - 경험 파일 내용 검토 후 수정

6. **자소서 작성 테스트**

   ```text
   [이름] 자소서 써줘: [문항] / [글자수]
   ```

## 결정 사항 및 배경

| 결정 | 이유 |
|------|------|
| 시간 분류를 연도로 | 학년은 범주가 달라 비교 어려움, 연도가 필터링에 유리 |
| 경험 데이터를 Skill이 아닌 md 파일로 | Skill은 절차용, 경험은 지식 레이어 |
| 범용 구조 (users/[이름]/) | 여러 사람이 쓸 수 있도록 설계 |
| 자소서 답변을 문단 형식으로 | 실제 자소서는 문단으로 작성됨 |
| 파싱 시 중복 처리 로직 추가 | 같은 경험이 여러 자소서에 반복 등장하기 때문 |
